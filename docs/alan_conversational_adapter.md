# Alan Conversational Telephony Adapter

## Overview
- Twilio handles inbound/outbound calls and streams audio through `<Start><Stream>` into the FastAPI service.
- The FastAPI adapter decodes Twilio media frames, performs streaming transcription with faster-whisper, applies turn-taking governance, and pushes voiced replies back through Twilio.
- ElevenLabs generates natural voice responses that are served to Twilio via short-lived, HMAC-signed playback URLs.
- Every call is logged in global and per-CallSid ledgers, capturing state transitions, recordings, file integrity metadata, and termination events.

## Environment Requirements
- Python 3.13 virtual environment located at `Agent X/.venv`.
- Install packages: `fastapi`, `uvicorn`, `python-multipart`, `requests`, `faster-whisper`, `twilio`, `elevenlabs`.
- Required environment variables (same session that launches uvicorn):
  - `TWILIO_ACCOUNT_SID`
  - `TWILIO_AUTH_TOKEN`
  - `TWILIO_FROM_NUMBER`
  - `TWILIO_VOICE_WEBHOOK`
  - `TWILIO_STATUS_WEBHOOK`
  - `ELEVENLABS_API_KEY`
  - `ELEVENLABS_VOICE_ID`
  - `PRESIGN_SECRET`

## Startup Workflow
1. Export the environment variables above.
2. Launch the service: `"C:\Users\signa\OneDrive\Desktop\Agent X\.venv\Scripts\python.exe" -m uvicorn service:app --host 0.0.0.0 --port 8000`.
3. Start ngrok: `ngrok http 8000` and update Twilio console URLs to the active host:
   - Voice URL: `https://<ngrok-host>/inbound`
   - Status Callback: `https://<ngrok-host>/status/callback`
   - Recording Callback: `https://<ngrok-host>/recording/complete`
4. Use `start-telephony.ps1` or the `telephony` alias for a one-word startup, optionally launching ngrok in a secondary window.

## Inbound Handling
- `/inbound` validates `CallSid`, responds with TwiML that greets the caller, opens the audio stream, and keeps the call alive with a long `<Pause>`.
- The `<Stream>` tag appends `callSid` as a query parameter so the WebSocket knows which call to govern.

## Streaming Transcription
- `/stream` accepts Twilio’s JSON frames, decodes base64 μ-law payloads into PCM, and feeds them to `WhisperStreamer`.
- `WhisperStreamer` flushes partial transcripts every ~0.25 seconds, maintaining overlap for context.
- Each transcript arrives as `(text, confidence)` for downstream gating.

## Turn-Taking Governance
- State machine cycles through `Listening → Transcribing → Hold/Reply` with logging:
  - Confidence threshold (default 0.7) filters low-certainty partials.
  - Silence gap threshold (default 0.8s) ensures the caller has paused.
  - Word-count threshold (default 3 words) avoids trivial responses.
- When all gates pass, the adapter invokes `aqi_generate_reply`, logs the transition, and moves back to `Listening` after speaking.

## Reply Generation & Playback
- Replies feed through the ElevenLabs helper which POSTs to the voice API, saves the MP3 under `recordings/<CallSid>/tts_<timestamp>.mp3`, and logs the save.
- Playback uses `<Play>` pointing at a presigned URL generated by `make_presigned_url(call_sid, filename)`.

## Presigned Playback Security
- `/presigned/{call_sid}/{filename}` validates expiry and HMAC signature before streaming the audio.
- Links default to a 60-second lifetime and return HTTP 403 when expired or tampered.
- Static files are mounted under `/recordings` but generally accessed via presigned URLs.

## Recording Capture & Integrity
- `/recording/complete` downloads Twilio recordings into `recordings/<CallSid>/RecordingSid_timestamp.wav`.
- Integrity metadata captured per file:
  - Duration (seconds)
  - Size (bytes)
  - SHA-256 hash
- All recordings log to both the global ledger and the per-call timeline.

## Logging & Audit Trail
- `log_event(event_type, call_sid, details)` writes to:
  - Global ledger: `recordings/log.csv`
  - Per-call timeline: `recordings/<CallSid>/timeline.csv`
- Events captured include:
  - `StateTransition`: every state change with reason
  - `Recording`: file saved with duration, size, hash
  - `NormalTermination`: orderly stream closure
  - `WatchdogCleanup`: idle timeouts handled by the watchdog thread
- Watchdog thread cleans idle call state after configurable inactivity (default 30s).

## Conversational Flow Summary
1. Twilio streams audio to `/stream`.
2. Whisper produces partial transcripts.
3. State machine decides whether to Hold or Reply.
4. AQI generates response text.
5. ElevenLabs renders audio, saved locally.
6. Presigned URL delivered to Twilio `<Play>`.
7. Call recordings download, hashed, and logged.
8. Ledgers track every transition and artifact across global and per-call scopes.

## Sequence Flow Summary
- Caller dials Twilio number.
- Twilio hits /inbound, receives TwiML with <Start><Stream> and long Pause.
- Twilio opens WebSocket to /stream and streams base64 μ-law frames.
- Adapter decodes frames, feeds Whisper streamer, emits partial transcripts with confidence scores.
- State machine evaluates confidence, silence gap, and transcript length to decide Hold or Reply.
- On Reply, AQI generates text; ElevenLabs renders MP3 saved under recordings/<CallSid>/; presigned URL is issued.
- Twilio REST update replaces live TwiML with <Play> pointing to presigned URL; caller hears response.
- Twilio recording callback downloads call audio, computes size/hash, logs to global ledger and per-call timeline.
- Watchdog cleans idle streams; ledgers capture StateTransition, Recording, NormalTermination/WatchdogCleanup events.
